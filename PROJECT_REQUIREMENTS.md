### 2.3 Pace Definitions

**Easy Pace:** 6:15-6:45/km
- Should be able to speak in short sentences
- Nose breathing possible
- Feels sustainable for hours
- Heart rate: ~65-75% max
- Purpose: Aerobic base building, recovery

**Tempo Pace:** 5:40-5:50/km
- "Comfortably hard"
- Can speak a few words but not sentences
- Sustainable for 20-30 minutes
- Heart rate: ~80-85% max
- Purpose: Lactate threshold training

**Interval/Speed Pace:** 5:10-5:30/km
- Hard effort, breathing labored
- Cannot speak
- Sustainable only in short bursts
- Heart rate: ~90-95% max
- Purpose: VO2 max improvement, speed development

**Race Pace:** 5:20/km
- Goal race effort
- Hard but sustainable for 30 minutes
- Mental familiarity critical
- Purpose: Race-specific conditioning

### 2.5 Singapore Environmental Considerations

**Heat/Humidity Challenges:**
- Singapore climate: 28-32¬∞C, 80-90% humidity year-round
- Heat builds quickly after 7am
- Dehydration risk high

**Mitigation Strategies:**
- **Timing:** All runs before 7am (early morning mandatory)
- **Hydration:** Evening before + water ready post-run
- **Route:** Use shaded park connectors when possible
- **Pacing:** Easy runs may be slower in heat - that's okay
- **Recovery:** Extra attention to hydration on rest days

## 3. Success Metrics & Philosophy

### 3.1 Three-Tier Success Framework

**Tier 1 Success (The Real Win) - MOST IMPORTANT:**
- ‚úÖ Completed 45+ of 56 scheduled runs (80% completion rate)
- ‚úÖ Ran 4x/week for at least 11 of 14 weeks
- ‚úÖ Proved to yourself you can commit
- **Why This Matters:** This IS the goal - everything else follows

**Tier 2 Success (Fitness Goal):**
- ‚úÖ Finished the 5.6km race without walking
- ‚úÖ Beat equivalent 2024 pace (5:48 ‚Üí anything faster for 5.6km)
- **Why This Matters:** Validates training worked

**Tier 3 Success (Bonus):**
- ‚úÖ Sub-30 minutes (5:20/km pace)
- **Why This Matters:** Exceeding expectations

**Critical Understanding:** If Tier 1 is achieved, Tiers 2 and 3 happen automatically. The discipline creates the fitness creates the time.

### 3.2 Race Day Strategy

**Target Splits for Sub-30min:**
- **Km 1:** 5:25 (start controlled, don't go out too fast - adrenaline trap)
- **Km 2:** 5:20 (settle into rhythm)
- **Km 3:** 5:20 (hold steady, this is where mental game matters)
- **Km 4:** 5:15 (start pushing if feeling good)
- **Km 5:** 5:15 (maintain pressure)
- **Last 600m:** Give everything (empty the tank)
- **Total Target:** ~29:45 (15-second buffer under 30:00)

**Pre-Race Protocol:**
- Wake 2-3 hours before race
- Light breakfast (familiar food - banana, toast, coffee)
- Arrive 45 minutes early
- 10-minute easy warm-up jog
- 3-4 strides at race pace
- Hit the toilet
- Start watch, trust training

**Mental Approach:**
- First km will feel easy (it should - be patient)
- Km 2-3 is mental - trust the pace
- Km 4-5 is when training pays off
- Last 600m is pure will - dig deep

### 3.3 What Success Really Means

**Reframing "Respectable":**
- Only comparison points that matter:
  1. You in 2024 (58 min 10km)
  2. You right now (5 mins at 6:30/km)
  3. You on April 16 (hopefully sub-30)
- Chasing "respectable" is a trap - there's always someone faster
- Sub-30 finish = top 25-30% of participants = objectively impressive
- More importantly: Came back after year off + committed 14 weeks + stuck to routine

**The Real Achievement:**
- Not about the 29:45 finish time
- About proving you can do what you say you'll do
- About building discipline you've struggled with
- About showing up 56 times when it would be easier not to
- The time is just the scoreboard

```javascript
function getCurrentWeek() {
  const today = new Date();
  const startDate = new Date('2026-01-13');
  const diffDays = Math.ceil((today - startDate) / (1000*60*60*24));
  const week = Math.ceil(diffDays / 7);
  return Math.max(1, Math.min(14, week));
}
```

## 8. Edge Cases & Error Handling

### 8.1 Data Validation

**Mobile App:**
- Required fields enforced (HTML5 validation)
- Number inputs reject non-numeric
- Date inputs provide picker
- Pace calculation handles division by zero (returns null)
- Empty states for no data scenarios

**User Errors:**
- Missing a run: Can log retroactively (date picker allows past dates)
- Wrong data entered: No edit functionality (keep simple) - re-log with correct data
- Lost data: No recovery if LocalStorage cleared - warning in docs

### 8.2 Week Transitions

**Automatic Week Detection:**
- App calculates current week based on date
- Week 1 = Jan 13-19
- Week 2 = Jan 20-26
- ...etc
- Week 14 = Apr 12-16

**Mid-Week Start:**
- User might start tracking in Week 2
- App still shows correct current week

### 8.3 Missed Runs

**Philosophy:**
- App doesn't judge - it tracks
- Logging missed runs is encouraged (honesty)
- Provides pattern recognition

**Workflow:**
- Select date
- Mark "No, missed it"
- Optional: Explain why
- Saves with completed=false
- Shows in history with red bar
- Still counts toward total logged days (transparency)

### 8.4 Time Zones

**Not Applicable:**
- User is in Singapore (fixed location)
- Race is in Singapore
- No travel planned during training
- All dates use device local time

### 8.5 Device Change

**No Cloud Sync:**
- Data trapped in browser LocalStorage
- No account system
- No export feature

**If User Gets New Phone:**
- Data lost
- Intentional design choice (simplicity)
- User can take screenshots as backup
- Excel tracker is separate backup

## 16. Coaching Philosophy Integration

### 16.1 Core Principles Embedded in Design

**1. Discipline Over Performance:**
- Success Tier 1 = consistency (45+ of 56 runs)
- Success Tier 3 = race time (bonus)
- Weekly reflections focus on habits, not speed
- Missed run logging = accountability without judgment

**2. Honesty & Self-Awareness:**
- "Why did you miss it?" field
- "What almost stopped me?" weekly question
- Feeling scale (Great/Good/Okay/Tough)
- Purpose: Pattern recognition, not guilt

**3. The 2-Day Rule:**
- Not enforced by app (user responsibility)
- Mentioned in documentation
- Weekly reflection prompts self-checking
- App provides data, user provides discipline

**4. Progress Visualization:**
- Streak counter (positive reinforcement)
- Week completion % (clear target: 75%+)
- Total runs (cumulative progress)
- All designed to show: "You're doing it"

**5. Reframing Success:**
- Dashboard badge: "On Track" not "Good/Bad"
- No comparison to other runners
- Focus on personal baselines
- Race time is scoreboard, not the game

### 16.2 Psychological Safeguards

**Avoiding Negative Spirals:**
- Logging missed runs ‚â† failure prompt
- Weekly rating includes "Poor" without judgment
- No streak breaks (1-day gaps allowed)
- No punitive language

**Building Positive Momentum:**
- Green completed run bars (visual satisfaction)
- Increasing stat numbers (tangible progress)
- Reflections capture wins (what worked well)
- Race countdown builds anticipation
